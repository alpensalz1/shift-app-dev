## シフト希望提出の仕様（`/shifts` ページ）

### ⚠️ 重要：役職によって提出方法が全く異なる

---

### アルバイトの場合

- **提出内容**: 「出勤できる日」を選ぶ
- **デフォルト**: 何も選択されていない状態からスタート
- **操作**: タップして出勤希望日を追加する（選択式）
- **時間入力**: 各日付に開始・終了時間を任意で入力可能
- **テーブル**: `shift_requests` に挿入

---

### 社員・役員の場合

- **提出内容**: 「休む日」を選ぶ（残りの日は全て出勤）
- **デフォルト**: 何も選択されていない状態からスタート
- **操作**: タップして休日を追加する（選択式）
- **テーブル**: `off_requests` に挿入

#### 休みパターン（必ずどれか1つを選択）

| パターン | 休み日数 | off_requests の type | 説明 |
|---|---|---|---|
| `5日休み` | 5日 | `休み` | 5日の休日を選択、出勤日は通常シフト |
| `6日休み` | 6日 | `休み` | 6日の休日を選択、出勤日は通常シフト |
| `5日休み ＋ 仕込みのみ` | 5日 | `仕込みのみ` | 5日休み、出勤日は仕込みシフトのみ |
| `5日休み ＋ 営業のみ` | 5日 | `営業のみ` | 5日休み、出勤日は営業シフトのみ |

**ルール**:
- 選択できる休日数はパターンで固定（5日 or 6日）
- 指定した日数に達したらそれ以上は選択できない
- 提出後は out_requests テーブルに日付ごとに1行ずつ保存される
- type フィールドで出勤日のシフト種別も同時に表現する

---

## 提出期間の仕組み（`getSubmissionPeriod`）

- 半月単$��で管理（前半: 1〜15日、後半: 16〜末日）
- 当月5日以前 → 当月後半（16日〜末日）のシフトを提出
- その他 → 翌月前半または後半のシフトを提出
- 締切日は提出期間の開始5日前

## データベーステーブル構成

### `shift_requests`（アルバイトの出勤希望）
```
staff_id: number
date: string (YYYY-MM-DDT00:00:00)
start_time: string | null
end_time: string | null
status?: 'pending' | 'rejected'
```

### `off_requests`（社員・役員の休み希望）
```
staff_id: number
date: string (YYYY-MM-DD)
type: '休み' | '仕込みのみ' | '営業のみ'
```

### `shift_fixed`（確定済みシフト）
```
date: string
shop_id: number
type: '仕込み' | '営業'
staff_id: number
start_time: string
end_time: string
```

## UI ガイドライン

- モバイルファーストデザイン（スマホでの操作を優先）
- Tailwind CSS + shadcn/ui を使用
- カレンダーは7列グリッド（日〜土）
- 日曜: 赤文字、土曜: 青文字
- アニメーション: `animate-fade-in`, `animate-slide-up`
- カラーテーマ:
  - アルバイト: blue/indigo グラデーション
  - 社員・役員（休み選択）: indigo/purple グラデーション
