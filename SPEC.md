## システム概要

- アプリ名：タナカたなか シフト管理
- 店舗：下北沢・三軒茶屋の2店舗
- 一査はシステム管理者であり、役員として登録されている

---

## 雇用形態（employment_type）

| 値 | 説明 |
|---|---|
| アルバイト | アルバイトスタッフ |
| 社員 | 正社員 |
| 役員 | 役員・マネージャー（一査を含む） |

⚠️ 「長期」という雇用形態は存在しない。コード上にある場合は削除すること。

- 営業：17:00 〜 24:00

### 三軒茶屋

## シフト希望提出の仕様（/shifts ページ）

⚠️ 重要：雇用形態によって提出方法が全く異なる

### アルバイトの場合

- 提出内容：出勤できる日と時間を選ぶ
- デフォルト：何も選択されていない状態からスタート
- 操作：タップして出勤希望日を追加し、**開始・終了時刻を必ず入力する（必須）**
- 時間入力：
  - 15分刻み（14:00, 14:15, 14:30 ... 24:00）
  - 有効範囲：14:00 〜 24:00
  - 終了時刻 > 開始時刻 であること（範囲外・逆転はシステムが弾く）
  - 最低勤務時間の制限なし
- テーブル：shift_requests に挿入

### 社員・役員の場合

- 提出内容：カレンダーで日付ごとに状態を選択する
- **デフォルト（未選択の日）= フル出勤（仕込み＋営業）**
- 各日付に設定できる3択：
  1. 休む
  2. 仕込みのみ入る
  3. 営業のみ入る
- 選択していない日はすべて「仕込み＋営業フル出勤」とみなす
- テーブル：off_requests に挿入（フル出勤の日は記録不要）

---

## 提出期間の仕組み

| 提出タイミング | 対象期間 |
|---|---|
| その月の **5日まで** | その月の **16日以降**（後半） |
| その月の **20日まで** | 翌月の **1〜15日**（前半） |

- 半月単位で管理（前半：1〜15日、後半：16〜末日）
- 締切日は提出期間の開始5日前

## データベーステーブル構成

### shift_requests（アルバイトの出勤希望）
- staff_id: number
- date: string (YYYY-MM-DDT00:00:00)
- start_time: string（必須）
- end_time: string（必須）
- status?: 'pending' | 'rejected'

### off_requests（社員・役員の休み・シフト種別希望）
- staff_id: number
- date: string (YYYY-MM-DD)
- type: '休み' | '仕込みのみ' | '営業のみ'

### shift_fixed（確定済みシフト）
- date: string
- shop_id: number
- type: '仕込み' | '営業'
- staff_id: number
- start_time: string
- end_time: string

## UI ガイドライン

- モバイルファーストデザイン（スマホでの操作を優先）
- Tailwind CSS + shadcn/ui を使用
- カレンダーは7列グリッド（日〜土）
- 日曜：赤文字、土曜：青文字
- アニメーション：animate-fade-in, animate-slide-up
